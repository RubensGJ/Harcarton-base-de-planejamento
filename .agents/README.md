## Ordem recomendada de uso

1. Use o `05-copiloto-principal-hacarthon.md` como ponto de entrada normal. Ele planeja, destrava, executa e decide quando chamar os outros agentes.
2. Use o `02-especialista-car-desafios.md` para enquadrar o problema no CAR, SICAR e no desafio escolhido.
3. Use o `01-guardiao-personas-raimundo-luana.md` para testar se Raimundo e Luana realmente seriam atendidos.
4. Use o `03-monitor-hackathon.md` para transformar a ideia em entregáveis, pitch, protótipo e cronograma.
5. Use o `04-conselheiro-sincero.md` para receber um veredito frio sobre viabilidade, competitividade e risco de escopo.

## Regra central

Cada agente deve proteger o projeto contra uma falha diferente:

- Projeto sem direção, próximo passo ou execução concreta.
- Persona sem dor real.
- Solução sem aderência ao CAR e aos três desafios.
- Hackathon sem entrega competitiva.
- Ideia bonita, mas inviável ou fraca para julgamento.

Para o haCARthon, a solução precisa ser simples de demonstrar, aderente ao desafio, compreensível para usuário comum, útil para analista público e competitiva nos critérios oficiais: pertinência temática, relevância do problema, usabilidade/design e inovação/criatividade.
